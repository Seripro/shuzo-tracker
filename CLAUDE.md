# プロジェクト概要

松岡修造の移動経路を特定する（という建前の）温度差ランキングアプリ。
日本全国の昨日と今日の気温差を計算し、最も気温が上がった県と下がった県の2つだけを表示する。

# 技術スタック

- フレームワーク: Next.js (App Router)
- 言語: TypeScript
- スタイリング: Tailwind CSS
- 気象データ取得: Open-Meteo API
- DB: なし（都度APIフェッチして計算）

# 開発ルール・制約

- バックエンドやDBは構築せず、外部APIへのリクエストと差分計算の処理はNext.jsのサーバーサイドで行う。
- ハッカソン用のプロトタイプであるため、過剰な抽象化や複雑なディレクトリ構成は避ける。
- クライアントには計算済みの結果（最高・最低の2県分）のみを渡すシンプルな構成にする。
